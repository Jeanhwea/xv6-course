* 硬件中断
1. 主板上的设备会产生中断, 常见的中断有
   - 用户键盘输入
   - 读写磁盘存储
   - 时钟中断
2. xv6 需要设置中断处理程序，来保证操作系统可以正常运行
   - 中断处理流程和系统调用过程类似
   - 大体流程是：主板硬件产生一个信号，我们对设备编程产生中断，然后 CPU 接受中断
3. 可以理解中断是来自硬件的

* PIC 中断
1. PIC 早期单核中断处理单元
   - 处理程序位于 picirq.c
   - 早期单个 PIC 最多支持 8 个中断，受限与设备引脚数量
     - 主 PIC: IRQ0 ~ IRQ7
     - 从 PIC: IRQ8 ~ IRQ16
   - inb/outb 指令配置主 PIC 产出 IRQ0 ~ IRQ7
   - xv6 直接关闭 PIC
2. 多核处理中断分为两个部分
   - lapic
   - ioapic
3. Local APIC
   - 本地 APIC, 位于单个 CPU 核心
   - lapicinit
   - 内存映射方式操作，而不是 inb/outb
4. IO APIC
   - I/O 系统中断
   - 内存映射方式操作，而不是 inb/outb
   - ioapicinit
